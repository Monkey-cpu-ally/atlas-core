# ATLAS Memory Constitution

## Purpose
Define how ATLAS captures, organizes, links, updates, protects, and evolves knowledge over time.

## Core Principles
- Memory is permanent unless intentionally archived.
- Knowledge should be connected rather than isolated.
- Every important decision should preserve its reasoning.
- Corrections improve memory instead of replacing history.
- Previous versions should remain traceable.

## Memory Types
- Foundational Memory
- Project Memory
- Research Memory
- Design Memory
- Technical Memory
- Lessons Learned
- Historical Archive

## Memory Lifecycle
Capture → Verify → Organize → Link → Apply → Review → Improve → Archive

## Linking Rules
Every memory should reference related projects, disciplines, AI roles, and foundational documents whenever practical.

## AI Responsibilities
Ajani maintains execution and project knowledge.
Hermes maintains engineering, manufacturing, and design knowledge.
Minerva maintains scientific, historical, educational, and cultural knowledge.
The Council approves changes to foundational memory.

## Final Principle
The value of ATLAS is measured not only by what it knows today, but by how effectively today's knowledge improves tomorrow's decisions.
