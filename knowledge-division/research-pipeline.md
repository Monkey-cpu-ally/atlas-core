# ATLAS Research Pipeline

The Research Pipeline controls how information enters the ATLAS Knowledge Division.

The rule is simple:

> Information is not knowledge until it has been checked, organized, connected, and judged.

## Pipeline Overview

```text
1. Collect
2. Source Check
3. Extract
4. Verify
5. Compare
6. Connect
7. Judge
8. Improve
9. Store
10. Teach
11. Project Link
12. Council Review
```

## 1. Collect

ATLAS may collect information from:

- Books
- Research papers
- Official documentation
- Patents
- University materials
- Museums and archives
- Standards organizations
- Manufacturer documentation
- Trusted engineering sources
- YouTube transcripts or summaries
- User-provided notes
- User-provided images
- GitHub repositories
- Public datasets
- Expert interviews or lectures

## 2. Source Check

Every source must be rated.

```text
A-Level: Primary source, peer-reviewed paper, official standard, official documentation
B-Level: University, textbook, reputable technical publication
C-Level: Established journalism, high-quality explainer, expert lecture
D-Level: Blog, forum, marketing page, social media
F-Level: Unsupported claim, hype, rumor, fake source, unclear origin
```

## 3. Extract

ATLAS extracts:

- Definitions
- Core mechanisms
- Materials
- Parts
- Process steps
- Costs
- Risks
- Limitations
- Claims
- Evidence
- Diagrams
- Historical context
- Related projects

## 4. Verify

ATLAS checks whether the claim is supported by multiple strong sources.

Verification categories:

```text
Unverified
Weakly Supported
Moderately Supported
Strongly Supported
Proven in Lab
Proven in Industry
Commercially Available
```

## 5. Compare

ATLAS compares the topic against:

- Competing technologies
- Older versions
- Modern versions
- Luxury versions
- Cheap generic versions
- Nature-based versions
- Fiction-inspired versions
- User's style preference
- Real manufacturing limits

## 6. Connect

Every topic must connect to related banks.

Example:

```text
Topic: Near-infrared industrial scanner
Primary Bank: Electrical Engineering
Related Banks: Physics, Software Engineering, Robotics, Manufacturing, Materials Science
```

## 7. Judge

ATLAS must give a direct judgment.

```text
Is it useful?
Is it ugly or beautiful?
Is it generic or original?
Is it realistic?
Is it too expensive?
Is it safe?
Is it hype?
Is it worth studying deeper?
```

## 8. Improve

Each AI gives an improvement pass.

### Hermes Improvement Pass
Engineering, buildability, materials, sensors, circuits, software, mechanical design, and manufacturing.

### Minerva Improvement Pass
Nature, biology, environmental value, medicine, plant systems, human use, beauty, story, and culture.

### Ajani Improvement Pass
Strategy, market, cost, competition, risk, ownership, deployment, protection, and long-term advantage.

## 9. Store

Stored knowledge must include:

- Topic name
- Bank
- Related banks
- Summary
- Source quality
- Evidence
- Risks
- AI notes
- Council status
- Project links
- Date added
- Date updated

## 10. Teach

ATLAS must be able to teach every topic at multiple levels:

```text
Child Level
Beginner Level
Student Level
Professional Level
Research Level
```

## 11. Project Link

Every useful knowledge entry should link to active ATLAS projects when relevant.

Examples:

- Weaver Project
- Green Robots
- Power Cell
- Resonance Scanner
- Mother Box concept
- Metal Flower system
- Hyper Axel
- Luxury brand/style engine
- ATLAS HUD
- Digital Twin Engine
- Quantum Simulator

## 12. Council Review

The Council must produce:

```text
Final Judgment:
Confidence:
Biggest Opportunity:
Biggest Risk:
Should Frazier Study This Further?
Should This Become a Project?
Next Action:
```

## Never Skip

ATLAS must never skip:

- Evidence check
- Weaknesses
- Risks
- Unknowns
- Source quality
- Cross-bank connections
- Best next step

## Output Standard

Every research answer should end with:

```text
Known Facts:
Assumptions:
Unknowns:
Risks:
Best Next Steps:
```
