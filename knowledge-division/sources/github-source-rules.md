# GitHub Source Rules

GitHub is one of the most important practical sources for ATLAS because it shows real code, real project structure, real documentation, real issues, and real collaboration.

## Mission

Use GitHub to help ATLAS learn software engineering, AI, robotics, electronics, CAD workflows, documentation habits, testing, deployment, and open-source project organization.

## GitHub Source Categories

```text
AI repositories
Robotics repositories
Software frameworks
Embedded systems
Simulation tools
CAD and open hardware
Educational repositories
Datasets
Documentation repositories
ATLAS user-provided repositories
```

## Trust Rule

A GitHub repository is not automatically trustworthy just because it has stars.

ATLAS must review:

- License
- Maintainer activity
- Documentation quality
- Test coverage
- Issue history
- Security warnings
- Release history
- Dependency health
- Code clarity
- Community reputation
- Whether the repository is archived or abandoned

## GitHub Source Passport Additions

Every GitHub source should include:

```text
Repository name
Owner
URL
License
Primary language
Last commit date
Stars / forks / issues if available
Archived status
Maintenance status
Documentation score
Testing score
Security notes
ATLAS use case
```

## Trust Levels

```text
A-Level: Official repository for a trusted project, strong documentation, active maintenance, tests, clear license
B-Level: High-quality educational or professional repo with good docs and active use
C-Level: Useful demo or example, but needs verification before production use
D-Level: Interesting but weakly maintained, unclear docs, or fragile code
F-Level: Unsafe, abandoned, malicious-looking, license unclear, or not worth using
```

## Approved Usage

ATLAS may use GitHub sources for:

- Learning project structure
- Studying implementation patterns
- Understanding APIs and frameworks
- Comparing design approaches
- Creating educational notes
- Identifying useful libraries
- Inspiring ATLAS architecture

## Restricted Usage

ATLAS must not:

- Copy code without respecting licenses
- Treat demo code as production-ready
- Use abandoned code blindly
- Trust security-sensitive code without review
- Assume popularity equals quality

## User GitHub Sources

Repositories shown or approved by Frazier receive special handling:

```text
Status: User-Approved Candidate
Action: Review, classify, summarize, and connect to Knowledge Banks
Write Access: Only with explicit user approval
Read Access: Allowed when provided or authorized
```
